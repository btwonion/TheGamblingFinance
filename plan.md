# TheGamblingFinance тАФ Implementation Plan

## Context

You want a self-hosted web app to track the finances of your poker nights with
friends: buy-ins from the bank, IOU-style chip trades between players, a
lifetime leaderboard across nights, and an end-of-night settlement view that
tells everyone exactly who pays whom. The repo at
`/home/antonw/development/TheGamblingFinance` is empty apart from `.git`, so
this is greenfield.

The pain the app removes: after a poker night, someone does mental math on
buy-ins and chip stacks and gets it wrong; someone else forgets a 5 тВм chip loan
and it festers. A proper ledger + a correct settlement algorithm fixes both.

**Confirmed decisions (from clarifying questions):**

| Decision | Choice |
|---|---|
| Auth scope | Self-hosted, multiple player accounts with one admin (you) |
| Database | PostgreSQL |
| Session scope | Multi-session history with lifetime leaderboard |
| Form factor | Mobile-first, desktop adapted |
| Trade model | IOU only тАФ no cash moves mid-game; all debts settle at night-end |
| UI tooling | Tailwind CSS + custom components (no component library) |
| Deployment | Docker Compose on a VPS |

---

## Stack

- **Backend**: Rust + **Axum** + **sqlx** (compile-time-checked queries) +
  **PostgreSQL**. Migrations via `sqlx migrate`. Passwords with **argon2id**.
  Auth via **httpOnly session cookies** backed by an `auth_sessions` table
  (safer than JWT-in-localStorage for a browser app). `tower-http` for CORS /
  compression / tracing; `tower-governor` for rate-limiting login.
- **Frontend**: Vue 3 (`<script setup>` + Composition API) + Vite + **Pinia** +
  **Vue Router** + **Tailwind CSS**. Axios with `withCredentials` for cookie
  auth. `openapi-typescript` to generate types from the backend's OpenAPI spec.
- **Deployment**: Docker Compose (services: `api`, `web`, `db`) behind nginx
  with Let's Encrypt.
- **Money**: `BIGINT` cents end-to-end; no floats anywhere.
- **Currency / locale**: EUR, German formatting (`de-DE`, `Europe/Berlin`).

## Database schema

UUID v7 PKs, `TIMESTAMPTZ` timestamps, `BIGINT` cents. Poker sessions are
called **nights** to avoid collision with auth sessions.

Tables (one migration per logical change):

- **`users`** тАФ id, email (unique), display_name, password_hash (argon2id),
  role (`admin`|`player`), created_at, updated_at, disabled_at. Admins disable,
  never delete (financial history depends on them).
- **`auth_sessions`** тАФ id, user_id, token_hash (SHA-256 of cookie value, raw
  never stored), created_at, expires_at, last_seen_at, user_agent, ip_addr,
  revoked_at. 30-day rolling expiry.
- **`nights`** тАФ id, created_by, title, played_on (DATE), currency (`CHAR(3)`
  reserved for future, defaults EUR), **`cents_per_chip`** (INTEGER, fixed
  per night), status (`open`|`closed`), opened_at, closed_at, notes.
- **`night_players`** тАФ (night_id, user_id) membership; composite PK.
- **`buy_ins`** тАФ id, night_id, user_id, amount_cents (>0), chips (>0),
  created_at, created_by. Composite FK to `night_players`.
- **`trades`** тАФ id, night_id, **`chip_giver_id`**, **`chip_receiver_id`**,
  chips (>0), amount_cents_owed (>0). Semantics: chips move giverтЖТreceiver;
  receiver owes giver `amount_cents_owed` at settlement. `CHECK (giver <>
  receiver)`. Both sides must be in `night_players`.
- **`cash_outs`** тАФ id, night_id, user_id, chips (>=0), created_at.
  `UNIQUE(night_id, user_id)` тАФ one cash-out per player per night.
- **`settlements`** тАФ night_id (PK), computed_at, algo_version. Materialized on
  close so historical reads stay cheap and answers stay authoritative even if
  the algorithm changes.
- **`settlement_balances`** тАФ (night_id, user_id, net_cents). Positive =
  receives, negative = pays.
- **`settlement_transfers`** тАФ id, night_id, from_user_id, to_user_id,
  amount_cents, seq (stable UI order).

Cascade policy: deleting a **user** is blocked (RESTRICT); deleting an open
**night** cascades to children; closed nights are not deletable via the UI.

Indexes: email (lower), auth expiry, nights.status, nights.played_on,
buy_ins/trades/cash_outs on (night_id), settlement_balances on user_id (hot
path for lifetime leaderboard).

## Settlement algorithm

Pure Rust function in `backend/src/domain/settlement.rs`:

```rust
pub fn settle(input: &SettlementInput) -> Result<SettlementOutput, SettleError>;
```

**Step 1 тАФ per-player net cents:**

```
cash_value(p)  = cash_outs[p].chips * night.cents_per_chip
buy_in_total(p)= sum of p's buy_ins.amount_cents
iou_received(p)= sum of trade.cents_owed WHERE trade.chip_giver    == p
iou_owed(p)    = sum of trade.cents_owed WHERE trade.chip_receiver == p
net(p) = cash_value(p) - buy_in_total(p) + iou_received(p) - iou_owed(p)
```

Invariant before emitting transfers: `sum(net) == 0`. If not, return
`SettleError::UnbalancedLedger { diff_cents }` тАФ the UI blocks close and shows
"Books off by 2,30 тВм тАФ check cash-outs".

**Step 2 тАФ greedy min cashflow:**

```
creditors = players with net > 0  (largest first)
debtors   = players with net < 0  (largest magnitude first)
while both non-empty:
    pay = min(-debtors[0].net, creditors[0].net)
    transfers.push(Transfer { from: debtors[0], to: creditors[0], amount_cents: pay })
    adjust both, drop any whose net is now 0
```

Produces тЙд NтИТ1 transfers; not provably optimal in general (NP-hard) but
optimal for home-poker shapes and explainable to humans. Tie-breaking by
`PlayerId` byte order ensures determinism so the stored `seq` is stable.

Edge cases:
- Single player тЖТ empty transfers (valid).
- Missing cash-out тЖТ reject close with actionable error.
- Self-trade тЖТ rejected at DB level.

**Close endpoint** wraps everything in a serializable transaction so two
racing admin clicks produce one winner + one 409.

## Style guide

Lifted verbatim into `docs/style-guide.md` and `frontend/src/styles/tokens.css`.

### Design philosophy

Dark, high-contrast, deliberate. Casino-lounge warmth (gold accents, saturated
greens/reds) grafted onto modern fintech discipline (tabular numerals, tight
hierarchy, generous whitespace). **No gradients, no glassmorphism, no
"playful" animations on money values** тАФ numbers must feel authoritative.

Three principles:
1. **Numbers are sacred** тАФ tabular, never italic, never truncated without a
   tooltip. Red/green only for deltas, never for labels.
2. **Touch first** тАФ primary actions thumb-reachable on phones; тЙе44├Ч44 px hit
   targets; FABs sit above the bottom nav.
3. **State is legible** тАФ open vs closed nights differ by pill color + border;
   sign of an amount, not just its color, conveys meaning.

### Typography

- UI: **Inter variable** (self-hosted as woff2 under
  `frontend/public/fonts/`).
- Money / chip counts / ranks: **Inter** with
  `font-feature-settings: "tnum" 1;` тЖТ utility class `.tabular`. No JetBrains
  Mono shipped (Inter's `tnum` is sufficient and saves bundle size).
- Scale (rem @ 16 px root): `xs .75 / sm .875 / base 1 / lg 1.125 / xl 1.25 /
  2xl 1.5 / 3xl 1.875 / 4xl 2.25`. Weights 500тАУ800 per role.
- Never italic. Negatives render as `-12,50 тВм` (minus, not parenthesis).

### Spacing, radius, elevation

Tailwind 4-pt base grid. Cards `p-4` mobile / `p-6` desktop. Inputs `h-11`
(44 px). Radii: inputs `rounded-md`, cards `rounded-lg`, bottom sheets
`rounded-t-xl`.

Dark UIs can't lean on shadow for hierarchy. Combine: surface-color steps
(`bg` тЖТ `surface` тЖТ `surface-2`) + 1 px `border-border` hairline +
`shadow-1` on cards for tactility. Bottom sheets get `shadow-sheet`.

### Component patterns

- **Buttons**: `primary` (filled emerald, `--primary-ink` text), `ghost`
  (bordered), `danger` (outline red), `icon` (circular 44 px). All get
  `focus-visible` ring in `--focus`.
- **Inputs**: `bg-surface-2 border-border` тЖТ `focus:border-border-strong`,
  error `border-negative`, helper text `text-xs text-ink-muted`.
- **Cards**: `bg-surface border border-border rounded-lg shadow-1`.
- **Stat tile**: tiny uppercase label, large tabular value, color by sign.
- **List row**: 48 px min height, border-b, tappable whole row.
- **Leaderboard row**: rank pill (#1 gets gold tint), avatar, name, trailing
  amount.
- **Bottom sheet**: drag handle, enters with `translateY 100% тЖТ 0` over
  `--dur-med`.
- **Status pill**: `open` uses `bg-positive-soft text-positive`; `closed` uses
  `bg-surface-2 text-ink-muted`.

### Motion

No page transitions. Hover тЖТ `--dur-fast`. Sheet / toast / dialog enter тЖТ
`--dur-med` ease-out; exit is half that. 200 ms color crossfade only when a
money sign flips. All suppressed under `prefers-reduced-motion`.

### Accessibility

WCAG 2.1 **AA**. Focus rings always visible (no global `outline:none`). All
interactive elements keyboard-reachable in visual order; bottom sheets trap
focus. Icon-only buttons carry `aria-label`. Toasts are live regions
(`role="status"`); errors `role="alert"`. Sign + label always carry meaning,
not just color. Inputs `font-size: 16px` to disable iOS zoom. `lang="de"` on
root; dates/currency via `Intl`.

### Shared contracts (locked down in Phase 0, changed only via explicit handshake)

1. **Migrations** at `migrations/*.sql` тАФ numbered by timestamp + initials.
   Never edit in place; new migrations only.
2. **API contract** `docs/contracts/openapi.json` тАФ hand-written skeleton day

4. **Settlement algorithm** тАФ pseudocode in `docs/adr/0002-iou-model.md` is

   backend's response.
5. **Agent briefing** `docs/agents.md` тАФ ownership table, branch rules,
   Conventional Commit scopes, PR checklist, escalation path.

### Sequencing

- **Phase 0 (serial, single agent):** scaffold repo tree; write migrations,
  OpenAPI skeleton, tokens.css, style-guide.md, agents.md, ADRs; seed
  `backend/Cargo.toml` and `frontend/package.json`; stub Dockerfiles that
  compile. Merge to `main`.
- **Phase 1 (parallel ├Ч 3):** Backend-Core, Backend-Auth, Frontend-Shell тАФ
  each on its own branch, depending only on Phase 0. Frontend-Shell mocks
  auth API via MSW to proceed without Backend-Auth being done.
- **Phase 2 (parallel ├Ч 2):** Frontend-Features (needs Phase 1 shell + core
  endpoints), DevOps (needs builds green). Start once Phase 1 merges land.
- **Phase 3 (serial):** integration тАФ boot Docker Compose, Playwright happy
  path, fix seams; polish (empty states, skeletons); seed script (admin + 2
  demo players); final README.

### Pitfalls and mitigations

| Pitfall | Mitigation |
|---|---|
| API contract drift | Backend CI regenerates `openapi.json`; frontend CI regenerates `types/api.ts`; PR fails on diff. |
| Duplicated frontend utilities | Frontend-Shell owns all `composables/` + components; Frontend-Features files `shell:gap` issues rather than adding its own. |
| Settlement algorithm divergence | Frontend never reimplements тАФ it renders the API. A frontend unit test loads backend fixture JSON to assert parity. |
| Migration number collisions | Filename = `YYYYMMDDHHMMSS_<initials>_<slug>.sql`; CI rejects duplicates. |
| Hardcoded hex colors | ESLint `no-restricted-syntax` blocks inline hex outside `styles/`. |
| Out-of-lane edits | Every PR body lists files + agent; CI warns on boundary crossings; humans review those PRs closely. |
| Missing login rate limiter | Backend-Auth PR must include a 429-after-N-fails integration test. |

Branching: `<agent>/<feature>` off `main`, rebased daily, squash-merged.
Commits Conventional: `feat(backend/nights): ...`, `feat(frontend/shell): ...`.

## Risks and assumptions (flag any you want to revisit)

1. **Chip rate fixed per night** (`nights.cents_per_chip`). Simpler, matches
   home-poker norms. If per-buy-in rates are ever needed, a future migration
   adds `cents_per_chip` to `buy_ins`.
2. **Every player has an account** тАФ no guest / walk-in players. If needed
   later, add `night_players.guest_name TEXT NULL` and allow null `user_id`.
3. **Admin creates all accounts** (temp password, user prompted to change on
   first login). No self-registration, no invite links.
4. **IOUs settle within a night only** тАФ no cross-night persistent debts.
5. **EUR only** (column reserved for future). German formatting.
6. **Postgres backup:** nightly `pg_dump` via a Compose service тЖТ host
   `/var/backups/gamblingfinance/`, rotated 14 days. Off-host replication is
   out of scope for MVP.
